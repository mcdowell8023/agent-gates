## 1. 问题陈述

agent-gates 现在是「commit 质量检查器」，不是「全流程编排器」。前段（需求→方案→计划）只软约束、后段（部署→自测）空白。本版补前段最高价值缺口：方案门控。

**v0.3 新认知**：方案门控要求"方案过异构审查"，但**异构审查链路本身在坏**（opencode 长 prompt 卡死、输出计划不出结论；codex 命令文档过时）。门控建在坏链路上 = 建在沙子上。所以**先修审查可靠性（前置），再建方案门控（主体）**。

## 2. 目标

把"写代码前必须对'要不要方案'做一次显式决策，需要方案时方案必须过异构审查"从软约束升级为硬门控，且：
- **三态路由**：按认知风险（不是 diff 大小）分流——琐碎自动放行、危险强制方案、中间问用户。
- **前置修好审查链路**：超时防卡死、命令分场景、消除过时文档。
- **Path A + Path B 都覆盖**。

非目标：部署/自测门控（下一版）、CI 服务端强制（独立任务，§3.6 说明）。

## 3. 设计

### 3.0 前置块：交叉审查工具可靠性修复

> 这是 v1.11.0 的**前置依赖**，先做先验证。下面是经 evidence 核对的真实命令形态。

**codex（0.134.0）两个子命令，分场景**：

| 场景 | 命令 |
|------|------|
| 审 git 代码变更（diff/commit） | `codex review --commit HEAD`（或 `--base <branch>` / `--uncommitted`；可带 `[PROMPT]` 定制审查指令）。自带 sandbox + 结构化输出 |
| 审任意内容（方案文档/设计问题/非 diff） | `codex exec -s read-only --skip-git-repo-check -C <dir> < prompt-file`。prompt 走 stdin |

清除：`codex -q --full-auto`（0.134.0 无 `-q`）、`codex-companion.mjs task --write`（OMC 插件老路径，过时）。

**opencode 失败模式全集 + 防护**：

| 失败 | 现状 | v1.11.0 修复 |
|------|------|-------------|
| P1 serve 泄漏 | `oc-reaper` ✅ | — |
| P2 exit 0 空输出 | `oc-review` retry ✅ | — |
| 长 prompt 卡死（无内置超时） | ❌ | **`timeout` 外层包裹**（默认 ~300s）+ **短 prompt 策略**：prompt ≤200 字，背景让模型从文件/代码自己读，不把大块内容塞进 prompt |
| 输出计划不出结论（gpt-5.5 当编排器） | ❌ | **无效输出检测**：结果不含预期结构（如 VERDICT / 实质结论）视为失败 → fallback codex |

**`agent-gates-review` 加固**（v1.10.0 现状：opencode 走位置参数、无超时）：
- 所有 opencode/codex 调用用 `timeout` 包裹，卡死即超时 → fallback 下一路由。
- `--plan` 模式审方案文档（长内容）：**不把方案全文塞进 prompt**，而是把方案文档路径给审查者，让它自己读（codex exec 在 `-C <dir>` 下能读；opencode 短 prompt + 文件引用）。这正是规避长 prompt 卡死的关键。
- 无效输出检测：opencode 返回无 VERDICT/无实质结论 → 当失败处理（治失败模式 B）。

### 3.1 三态路由（核心，替代 v0.2 的二元阈值）

门控判断的是**认知风险**，不是 diff 大小。三态：

| 态 | 触发判定 | 处理 | gate 验什么 |
|----|---------|------|-----------|
| **auto-trivial** | **窄白名单**：纯文档(.md)、文案、补测试、样板/配置(无逻辑) | 自动放行 | 无（痕迹可选，auto-trivial 的意义是免打扰）|
| **requires-plan** | **危险类别启发式命中**：路径/内容涉及 auth/security/权限、数据库迁移(`migrations/`,`*.sql`,schema)、架构/跨模块、公共 API、并发 | **强制方案（带 PLAN_REVIEW），不接受豁免** | 方案存在 + PLAN_REVIEW 三件套 |
| **skip-with-user-approval** | 其余非 trivial（既不在白名单也未命中危险类别） | 问用户：要方案就写、不要就授权豁免 | 方案 **或** 豁免记录（二选一）|

**危险类别检测**（pre-commit，启发式）：
- 路径匹配：`migrations/`、`*.sql`、`*auth*`、`*security*`、`*permission*`、`*acl*`、schema 文件等。
- 这是启发式，有漏报/误报（见 §7 已知限制）。命中即 `requires-plan`（从严）。
- 用户选了"危险类别强制不可跳过"——命中后**豁免记录不被接受，只接受方案**。

### 3.2 决策痕迹（plan-decision artifact）+ 防伪（回应 codex C/D）

gate 不再猜"该不该有方案"，只验"决策做过"。三态各自的痕迹：
- requires-plan → `.agent/plans/` 下存在**任一** `.md` 带 `PLAN_REVIEW` 三件套（v0.4 简化：gate 不做 topic→staged 精确匹配，只查"有没有做过方案决策"。精确绑定留 v1.12）
- skip-with-approval → `.agent/plans/` 下存在**任一** `.skip.md`（豁免记录，命令生成）
- auto-trivial → 无痕迹要求（自动放行，gate 不查）

**豁免记录由命令生成**（防伪，回应 codex D）：
- 新增 `agent-gates-plan-decision`（或并入 `agent-gates-review`）：
  - `skip --reason "<理由>" --topic <name>` → 生成 `.skip.md`，自动写入 branch、HEAD、时间戳、任务摘要、`GENERATED_BY: agent-gates` 标记。
  - `trivial --topic <name>` → 生成 trivial 痕迹。
- gate 只认带完整机器生成元数据的记录，**手写格式不认**——抬高伪造成本。
- client 端无法根治伪造（agent 能调命令伪造），真防伪需 server-side（§3.6）。

### 3.3 机制一：PostToolUse 软提醒（`memory-reminder.mjs`）

新增检测器 `detectSourceEditWithoutDecision`：
- matcher 扩展加 `Edit|Write|edit|write`（+ install.sh matcher 迁移，回应 v0.2 #2）。
- 触发：写源码 + 命中扩展名（排除 test）+ 当前 `.agent/` 存在但**无该任务的决策痕迹**（无带标记方案、无豁免记录、无 trivial 标记）。
- 动作：注入双语软提醒「开工前先对'要不要方案'做决策：危险改动(认证/数据/架构…)必须写方案过 `agent-gates-review --plan`；其余非 trivial 决定要方案或 `agent-gates-plan-decision skip`；trivial 可 `... trivial`」。
- 不阻断（PostToolUse 跨平台阻断不一致）。
- 静音：存在任一有效决策痕迹即止（不再"任意 .md 静音"，回应 D3）。

### 3.4 机制二：pre-commit 硬门控（CHECK 3，`agent-quality-gate.sh`）

加在 Gate 1 后、Gate 2 前。流程：
1. trivial 豁免（gate 顶部已有：无新增源码 + 小 diff → 整体 skip）。
2. 算态：危险类别启发式命中 → `requires-plan`；命中 auto-trivial 白名单 → 验 trivial 痕迹；其余 → `skip-with-approval`。
3. 验对应决策痕迹：
   - requires-plan：方案（Path A proposal / Path B `.agent/plans/*.md`）存在 + `PLAN_REVIEW` 三件套（L1+ 机器）。缺 → **fail（不接受豁免）**。
   - skip-with-approval：方案 **或** 命令生成的豁免记录，二者皆无 → fail。
   - auto-trivial：trivial 痕迹存在（或直接放行，痕迹可选）。
4. Path A 多 change 绑定（v0.2）：staged 落在 `openspec/changes/<X>/` 则查 X 的 proposal。
5. L0 降级（v0.4 明确语义）：无异构工具时方案**必须存在**（非空 `.md`），但**不要求** `PLAN_REVIEW` 标记。gate 放行 + 输出 warn "L0: 方案存在但无法证明过异构审查"。意图：有方案总比没有强；L0 用户升级工具后自然升级到标记要求。
6. 逃生阀 `SKIP_PLAN_CHECK=1`。

### 3.5 方案审查证据格式 + `agent-gates-review --plan`

```
<!-- PLAN_REVIEW: L1 -->
<!-- PLAN_REVIEW_TOOL: codex -->
<!-- PLAN_REVIEW_MODEL: gpt-5-series -->
```

`agent-gates-review --plan <plan-file>` 原生输出三件套（回应 D2），审方案文档走 §3.0 加固后的链路（codex exec / 短 opencode + 超时）。gate 只认三件套齐全。

### 3.6 造假风险与诚实声明

client-side git hook 可被绕过：`--no-verify`、改 `core.hooksPath`、直接覆盖 `.githooks/`。三态/决策痕迹/命令生成都只**抬高摩擦**，不根治。

**诚实声明**：本门控是**减速带**。真约束是纵深防御第二层——**CI 跑同款检查 + branch protection 拒未过 CI 的 PR**。client hook 与 CI 互补，非替代。（CI 模板列入后续。）

## 5. 测试计划（TDD）

**前置块**：
- agent-gates-review timeout 生效（mock 卡死进程→超时 fallback）；--plan 输出三件套；无效输出（无 VERDICT）→fallback。
- agent-gates-plan-decision: skip/trivial 生成痕迹带元数据；手写格式 gate 不认。

**三态 CHECK 3**（run_gate.sh）：
- requires-plan: 危险路径(`migrations/x.sql`)无方案→FAIL；有方案+三件套→PASS；危险类别**有豁免记录但无方案→FAIL**（不接受豁免）。
- skip-with-approval: 普通非 trivial 无痕迹→FAIL；有豁免记录→PASS；有方案→PASS。
- auto-trivial: 纯 .md/补测试→放行。
- 边界（v0.2）: 多 active change / 空 plan / 缺三件套 / 无 capability 文件 / 非常见扩展名 / L0 豁免 / SKIP_PLAN_CHECK / NEEDS_REVIEW 重构触发。

**PostToolUse**: 写 src 无痕迹→提醒；有痕迹→静音；非 src/无 .agent/→静音。
**matcher 迁移**: 老 matcher→升级含 Edit|Write；已新→幂等。
**回归**: 现有全套零回归。

## 6. 已知限制

- **危险类别启发式不可靠**：路径/关键词匹配有**漏报**（危险改动未命中关键词→降级到 skip，用户可能授权跳过）和**误报**（无害文件名含 auth）。漏报是主要风险。缓解：关键词清单可配置 + 文档提示用户对未命中的危险改动主动要方案。这是启发式的固有局限，非 bug。
- PostToolUse 无状态：无痕迹期间重复提醒，建立后即止。
- 造假/绕过：见 §3.6，减速带非锁。
- 三态复杂度：比二元高，危险类别清单需持续打磨。
